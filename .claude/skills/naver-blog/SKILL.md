# naver-blog — 기사 URL → 네이버 블로그 초안

## 트리거

- 명시 호출: `/naver-blog <URL>`
- 자연어: "이 기사 블로그 초안 써줘", "네이버 블로그용으로 정리해줘", "이 URL 블로그로 올리게 정리해줘"

## 입력 / 출력

| 항목 | 내용 |
|------|------|
| **입력** | 기사·웹페이지 URL 1개 (선택: 추가 메모, 톤 지시) |
| **출력** | `Q4-Naver-Blog/draft.md` (마크다운 초안) + 가능하면 네이버 블로그 작성창에 임시저장 |

## 작성 규칙 (톤 & 구조)

### 톤
- **친구한테 카톡으로 풀어주듯** 캐주얼한 반말체
- 기자처럼 객관적·중립적으로 쓰지 말 것
- 핵심을 짚고 나서 "솔직히 ~", "근데 진짜 문제는 ~" 같은 개인적 해석 섞기
- `핵심만 말하면, ~` 패턴을 도입·핵심·내 생각 섹션 중 최소 1~2회 사용 (말버릇)

### 구조 (반드시 이 4단)

```markdown
# {기사의 핵심을 잡은 한 줄 제목 — 의문/감탄 부호 OK}

## 도입
- 왜 이 기사를 봤는지, 어떤 맥락에서 흥미로웠는지 2~3문장
- 기존 통념 → 새 흐름 대비를 살리면 좋음

## 핵심 내용
- 기사의 팩트를 3~6개의 불릿으로 압축 (숫자·고유명사는 그대로)
- 비교표가 필요하면 학교/기업/시기별로 항목화
- 마지막에 전문가 코멘트나 기사 결론 한 줄 인용

## 내 생각
- 옛날과 지금의 차이, 누가 손해 보는지, 구조적 문제인지 개인 문제인지
- 감정 + 근거 둘 다 (그냥 무섭다 X → 왜 무섭다 O)

## 마무리
- 독자에게 던지는 질문 또는 작은 실천 제안
- "AI 매일 만지는 게 안 만지는 것보다 낫겠다" 류의 가벼운 결심
```

### 푸터 (필수)

```markdown
---

**원문:** [{매체명} — {기사 원제목}]({URL})
```

## SOP (실행 순서)

### 1) URL 분석
- WebFetch로 URL 본문 가져오기
- 추출할 것: 제목, 매체명, 핵심 팩트(숫자·고유명사·날짜), 인용문 1~2개
- 광고/추천기사 영역은 무시

### 2) 한 줄 헤드라인 결정
- 기사 제목을 그대로 쓰지 말 것. 사용자 입장에서 한 줄 후킹으로 재작성
- 길면 부제를 `—`로 붙이기 (예: `대학가에 번지는 'AI 디바이드' — 유료 구독이 성적까지 가른다고?`)

### 3) 4단 초안 작성
- 위 "구조" 그대로
- 핵심 내용 섹션의 숫자/고유명사는 **원문 그대로** 사용 (왜곡 금지)
- 출처가 불명확한 사실은 쓰지 말 것

### 4) 파일 저장
- 경로: `Q4-Naver-Blog/draft.md` (덮어쓰기)
- 작업 디렉터리가 다른 프로젝트면 사용자에게 저장 경로 확인

### 5) 네이버 블로그 임시저장 (Playwright MCP가 활성일 때만 시도)

```
.mcp.json에 playwright가 등록돼 있고, .env에 NAVER_ID/NAVER_PW가 있을 때만 시도.
없으면 5단계는 건너뛰고 .md 저장으로 마무리.
```

흐름:
1. `mcp__playwright__browser_navigate` → `https://nid.naver.com/nidlogin.login`
2. ID/PW 입력 시도 (`.env`의 `NAVER_ID`, `NAVER_PW`)
3. **CAPTCHA가 뜨면 즉시 중단** → 사용자에게 "캡차 떴어요. 수동 로그인 후 다시 호출해주세요" 알림. `draft.md`는 이미 저장돼 있음
4. 로그인 성공 시 `https://blog.naver.com/{NAVER_ID}?Redirect=Write` 이동
5. 제목 입력 → 본문 붙여넣기 → **임시저장 버튼** 클릭 (절대 "발행" 누르지 말 것)
6. 임시저장 완료 토스트 확인 후 스크린샷

### 6) 사용자에게 보고
- 저장된 `draft.md` 경로 명시
- 임시저장 성공/실패 여부
- 다음 액션 제안 ("발행 전에 직접 검토하세요")

## 안전 규칙

- **절대 "발행" 버튼 누르지 않기.** 임시저장까지만.
- `.env` 파일은 읽기만, 절대 출력에 노출 X
- CAPTCHA·2차 인증 → 즉시 중단, 사용자에게 핸드오프
- Playwright MCP가 실행 중일 때 사용자가 브라우저를 만지면 충돌 → "스크립트 동작 중엔 브라우저 건드리지 마세요" 안내
- 사실관계 왜곡 금지: 숫자·고유명사는 본문에서 그대로 옮기고, 추측은 "내 생각" 섹션에만 쓰기
