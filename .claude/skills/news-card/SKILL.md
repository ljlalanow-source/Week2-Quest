# news-card — 기사 URL → 인스타그램 뉴스 카드 (HTML + PNG)

## 트리거

- 명시 호출: `/news-card <URL>`
- 자연어: "뉴스 카드 만들어줘", "인스타용 카드 만들어줘", "이 기사 카드뉴스로 만들어줘", "이 URL로 뉴스 카드 제작"

## 입력 / 출력

| 항목 | 내용 |
|------|------|
| **입력** | 기사·웹페이지 URL 1개 (선택: 카테고리, 출력 폴더) |
| **출력** | `Q5-News-Card/output/newscard-YYYY-MM-DD.html` + 동일 이름의 `.png` |
| **크기** | 1080 × 1350 (인스타그램 피드 4:5 세로형) |

기본 출력 폴더는 `Q5-News-Card/output/`. 프로젝트 구조가 다르면 사용자에게 저장 경로 확인.

## 추출 규칙 (콘텐츠)

WebFetch로 본문을 가져온 뒤 다음 항목을 추출:

| 필드 | 길이 제한 | 작성 가이드 |
|------|---------|---------|
| **헤드라인** | 15자 이내 | 기사 제목 그대로 쓰지 말 것. 핵심을 한 줄로 후킹. 2줄로 분할 가능 |
| **서브타이틀** | 30자 이내 (2줄 분할 OK) | 헤드라인을 부연. 누가/왜 중요한지 |
| **핵심 포인트 3개** | 각 20자 이내 | **구체적 숫자·고유명사·인용** 우선. 추상적 표현 금지 |
| **카테고리** | 2~3 단어 | 예: "AI · 교육", "정치 · 외교", "경제 · 부동산" |
| **출처** | 매체명 | 예: "뉴시스", "한겨레", "조선일보" |
| **출처 로고** | 1자 | 매체명 첫 글자 (한글 한 글자 또는 영문 이니셜) |
| **날짜** | YYYY.MM.DD | 기사 게재일 (없으면 오늘 날짜) |

### 핵심 포인트 작성 원칙 (가장 중요)

- **숫자가 있으면 무조건 넣기** ("월 25만원", "33억원", "0원")
- **고유명사 비교 살리기** ("동국대 35만원 vs 고려대 0원")
- **인용문 짧게 따오기** (전문가 "새로운 정보 격차")
- ❌ 두루뭉술한 표현 금지: "성능 차이로 격차 발생", "정보 불평등 심화" 같은 추상어
- ✅ 강한 포인트 예시:
  - "학생 월 최대 25만원 AI 구독료"
  - "동국대 35만원 지원, 고려대 0원"
  - "전문가 '새로운 정보 격차' 경고"

### 사용자 컨펌 포인트

추출 직후 사용자에게 핵심 포인트 3가지를 보여주고 컨펌받기. 약하면 옵션 A/B/C로 재제안.

```
✅ 핵심 포인트 후보
1. {포인트 1}
2. {포인트 2}
3. {포인트 3}

이대로 카드 만들까요? 약하면 다른 안 뽑아드릴게요.
```

## 디자인 (애플 스타일 글래스모피즘)

기본 템플릿: [`templates/glass.html`](templates/glass.html)

### 디자인 특징
- 배경: 부드러운 파스텔 라디얼 그라데이션 (핑크/블루/민트/옐로우 4코너)
- 컬러 블롭 3개를 blur로 깔아 유리에 색이 비치도록
- 모든 카드: `rgba(255,255,255,0.55)` + `backdrop-filter: blur(30px) saturate(180%)`
- 헤드라인: 검정 텍스트, 100px, 자간 -0.045em
- 번호 배지 3개에 각기 다른 그라데이션:
  - 1: 핑크 → 퍼플 → 블루 (`#ff6b9d → #c084fc → #60a5fa`)
  - 2: 블루 → 민트 (`#60a5fa → #34d399`)
  - 3: 옐로우 → 오렌지 (`#fbbf24 → #fb923c`)
- 푸터 글래스 박스 안에 출처 로고/매체명/날짜 + SWIPE → 인디케이터

### 자체 완결성
- 외부 폰트/이미지/CDN 의존 X
- 시스템 폰트만: `-apple-system, "Pretendard", "Apple SD Gothic Neo", "Noto Sans KR"`
- 단일 HTML 파일로 어디서든 열리도록

### 사용자가 다른 톤 요청 시
- "다크 버전" → 배경 `#0a0a0f` + 라디얼 인디고/핑크로 변경, 텍스트 흰색
- "미니멀" → 블롭 제거, 흰 배경 + 그레이 톤 카드
- 그 외 → 기본 글래스 유지하고 색만 조정

## SOP (실행 순서)

### 1) URL 분석
- `WebFetch`로 본문 가져오기
- 본문이 짧으면 한 번 더 호출해서 구체 숫자/인용문 보강 요청
- 광고/추천기사/관련영상 영역 무시

### 2) 콘텐츠 추출 + 사용자 컨펌
- 위 "추출 규칙" 표대로 필드 채우기
- 핵심 포인트 3개를 사용자에게 보여주고 컨펌
- 약하다고 하면 옵션 A/B/C로 재제안 (숫자 중심 / 키워드 중심 / 인용 중심)

### 3) HTML 생성
- `templates/glass.html`을 Read로 읽기
- placeholder 치환 (Edit 또는 Write):
  - `{{PAGE_TITLE}}` — 헤드라인 한 줄 텍스트
  - `{{CATEGORY}}` — 카테고리 배지
  - `{{HEADLINE_HTML}}` — `<br>` 포함 헤드라인 (예: `유료 AI 구독이<br>대학생 성적 좌우`)
  - `{{SUBTITLE_HTML}}` — `<br>` 포함 서브타이틀
  - `{{POINT_1}}`, `{{POINT_2}}`, `{{POINT_3}}`
  - `{{SOURCE_LOGO}}` — 1글자
  - `{{SOURCE_NAME}}`
  - `{{DATE}}` — YYYY.MM.DD
- 저장 경로: `{출력폴더}/newscard-{기사날짜 또는 오늘}.html`

### 4) PNG 변환 (Playwright MCP)

**중요**: `file://` 프로토콜은 Playwright에서 차단됨 → 로컬 HTTP 서버 우회 필수.

```bash
# 1) HTTP 서버 백그라운드 실행
cd {출력폴더} && python3 -m http.server 8765 > /dev/null 2>&1 &
# PID 저장 (예: $!) — 나중에 kill하기 위함

# 2) 200 응답 확인
curl -s -o /dev/null -w "%{http_code}\n" http://localhost:8765/{파일명}.html
```

Playwright MCP 흐름:
1. `mcp__playwright__browser_resize` → width 1080, height 1350
2. `mcp__playwright__browser_navigate` → `http://localhost:8765/{파일명}.html`
3. `mcp__playwright__browser_take_screenshot` → `filename: {출력폴더}/{파일명}.png`, `fullPage: true`, `type: png`
4. `mcp__playwright__browser_close`
5. `kill {PID}` 로 HTTP 서버 종료

### 5) 검증 + 보고
- `Read`로 생성된 PNG를 시각 확인 (텍스트 잘림, 레이아웃 깨짐 체크)
- 사용자에게:
  - HTML 경로
  - PNG 경로 (1080×1350)
  - 다음 액션 제안 ("바로 인스타 업로드 가능")

## 안전 규칙

- **사실 왜곡 금지**: 추출한 숫자·고유명사는 기사 원문 그대로. 추측이나 윤색은 안 됨.
- **포트 충돌 주의**: 8765 포트가 이미 사용 중이면 8766, 8767로 순차 시도. 끝나면 반드시 kill.
- **이미지 인용**: 기사 본문 이미지는 저작권 문제로 절대 카드에 넣지 않음. 디자인은 글래스+텍스트만.
- **인스타 자동 업로드 X**: 이 skill은 PNG 생성까지만. 업로드는 사용자 수동.
- **글자 수 제한 엄수**: 20자 초과 시 카드 레이아웃 깨짐. 줄여서 재작성.
