# 부부 월간 보고서 (bubu-report)

부부가 **함께 앉아** 한 달을 의미 중심으로 결산하는 다이어리 생성 스킬. 작성자는 둘 중 한 사람이지만, 답변은 둘이 합의해서 채워나간다.

## 핵심 원칙

- **의미 중심 > 정보 나열.** "5/3 광장시장 갔음"보다 "온종일 웃었던 날" 같은 의미가 더 중요하다.
- **빈칸을 두려워하지 말 것.** 채우기 부담스러운 섹션은 `_(이번 달엔 없음)_`으로 비워둬도 OK. 단, 사진과 한 줄 정의는 **무조건 채우도록 독촉**한다.
- **거창할 필요 없음.** "그냥 같이 산책했다"도 베스트 모먼트가 될 수 있다.
- **부담스럽지 않은 페이스.** 한 번에 모든 걸 묻지 말고 묶음으로 2~3개씩 진행.

## 전제 정보

- **결혼기념일:** 2025-07-19
- **N개월차 계산 방식:** 만 계산 (결혼한 달 = 0개월차 = 신혼월)
  - 2025-07 → 0개월차
  - 2025-08 → 1개월차
  - 2026-05 → 10개월차
  - 2026-07 → 12개월차 (= 1주년)
  - 공식: `N = (year - 2025) * 12 + (month - 7)`
- **저장 위치:** 프로젝트 루트의 `reports/` 폴더
- **파일명:** `YYYY-MM.md` (예: `2026-05.md`)
- **누적 파일:**
  - `reports/_lifework-candidates.md` (인생작 후보 누적)
  - `reports/_couple-phrases.md` (부부 말버릇 누적)

## 워크플로우

### Step 0 — 자동 메타데이터 계산

스킬이 트리거되면 먼저 다음을 정리한다.

1. **대상 월 확인.** 사용자가 명시하지 않았다면: "어느 달 보고서를 만들까요? (기본: 지난달 = `YYYY-MM`)" 라고 한 번 묻고, 응답 없으면 지난달로 진행.
2. **N개월차 계산.** 위 공식으로 계산.
3. **작성일.** 오늘 날짜 (`date +%Y-%m-%d`).
4. **기존 파일 확인.** `reports/YYYY-MM.md`가 이미 있으면 → "기존 보고서가 있어요. 이어쓸까요(append/update), 새로 쓸까요(overwrite)?" 묻기. 덮어쓸 땐 기존 내용을 먼저 보여주고 확인받는다.

### Step 1 — 메모 덤프 받기

이렇게 운을 띄운다:

> "이번 달 두 분이 같이 떠올린 메모/대화/사진 자유롭게 던져주세요. 카톡 캡처, 캘린더, 일기, 그냥 머릿속 — 정리 안 돼도 OK. 없으면 바로 인터뷰로 넘어가도 됩니다."

사용자가 텍스트를 주면 Step 2로, "없어"라고 하면 Step 3로 바로 진행.

### Step 2 — 1차 분류

받은 메모를 템플릿의 어느 섹션에 들어갈지 자동 분류:

- 날짜 + 장소/활동 → 함께 한 활동 표
- "재밌었어", "좋았어" 같은 감상 → 베스트 모먼트 후보
- 영화/드라마/책 제목 → 콘텐츠 섹션
- "처음 ~했어" → Firsts
- "고마웠어", "미안해" 같은 말 → 서로에게 한 마디

분류 결과를 사용자에게 짧게 보여주고 ("이렇게 분류했어요, 맞나요?"), 채워지지 않은 섹션 목록도 같이 알려준다.

### Step 3 — 빈칸 인터뷰 (의미중심)

비어있는 섹션을 **묶음 단위**로 질문한다. 한 번에 1~2개씩만. 부담 주지 말 것.

순서는 다음을 권장하지만, 사용자가 채워온 항목은 건너뛴다:

**라운드 1 — 큰 그림 평가** (한 줄 정의 + 감정 온도를 묶어서. 둘 다 한 달 전체를 한 컷에 평가하는 항목이라 자연스럽게 이어진다.)
- "이번 달 우리는 ___였다. (한 줄로 정의)" ← 필수
- "이번 달 자기 점수, 둘이 각자 1~10점이랑 한 줄 사유. 남편 → 아내 순서로."

**라운드 2 — 대표 사진**
- "이번 달 대표 사진 한 장 — 경로/링크 알려주세요. 언제·어디서 찍은 건지, 왜 이걸 골랐는지도 한 줄로." ← **무조건 채우도록 독촉**. 사용자가 "아직 못 골랐어"라고 해도 "한 장만 같이 떠올려봐요. 카톡 보낸 사진 중에 아무거나라도 OK"처럼 한 번 더 권하기. 그래도 못 정하면 `_(고르는 중)_`으로 마커.

**라운드 3 — 베스트 모먼트**
- "기억에 남는 순간 3개. 거창할 필요 없어요. 산책도 OK."
- 각각: 날짜 / 제목 / 무슨 일 / 그때 했던 말이나 행동
- 3개 못 채우면 1~2개라도 진행.

**라운드 4 — 활동 & 콘텐츠**
- 활동 표가 비어있으면: "외출/데이트/집 이벤트 떠오르는 거 다 알려주세요. 표로 정리할게요."
- 콘텐츠: "이번 달 본 거 4종(영화/드라마/책/유튜브) — 없는 칸은 `-`로 비워도 OK."
- 콘텐츠 받을 때 마지막에 **꼭** 묻기: "이 중에 올해의 인생작 후보로 넣을 거 있어요? (누적됨)"

**라운드 5 — 서로에게**
- 남편 → 아내, 아내 → 남편 한 마디씩.
- 고마웠던 순간 1개씩.
- (둘이 같이 앉아있으니) "서로 보면서 직접 말해보고 그대로 받아적어도 좋아요."

**라운드 6 — Firsts & 깨달음**
- 처음 4종: 일/장소/음식/사람 — 없으면 `-`.
- 작은 깨달음 1개 + 다음에 시도할 것 1개.

**라운드 7 — 우리집 말**
- "이번 달 자주 한 말, 둘만의 말버릇 있나요?"
- 받은 것들은 메인 보고서에도 적고, 누적 파일에도 추가.

### Step 4 — 미리보기 → 저장

1. 전체 보고서를 화면에 미리보기로 보여준다.
2. "이대로 저장할까요? 수정할 부분 있으면 말씀해주세요." 묻기.
3. OK 받으면:
   - `reports/YYYY-MM.md` 저장 (없으면 `reports/` 폴더 생성)
   - 새 인생작 후보 → `reports/_lifework-candidates.md`에 append
   - 새 부부 말버릇 → `reports/_couple-phrases.md`에 append
   - 저장 후 파일 경로 알려주기.

## 산출물 형식

### 메인 보고서

[assets/template.md](assets/template.md)의 플레이스홀더(`{{...}}`)를 채워서 사용한다. 빈 섹션은 `_(이번 달엔 없음)_` 같은 마커로 명시.

### 누적 파일 형식

**`reports/_lifework-candidates.md`:**
```markdown
# 🏆 우리집 인생작 후보 누적

| 추가월 | 종류 | 제목 | 한줄 메모 |
|--------|------|------|-----------|
| 2026-05 | 🎬 영화 | ... | ... |
```

**`reports/_couple-phrases.md`:**
```markdown
# 💬 우리집 말버릇 누적

## 2026-05 (10개월차)
- "..." — 상황 한 줄
- "..." — 상황 한 줄
```

파일이 없으면 헤더부터 생성, 있으면 해당 월 섹션을 append.

## 트리거 예시

다음 같은 발화에 반드시 트리거되어야 한다:

- `/bubu-report`
- "이번 달 결산 도와줘"
- "5월 부부 일기 정리하자"
- "결혼 10개월차 보고서 만들어줘"
- "우리집 한 달 다이어리 쓰자"
- "월간 보고서"
- "5월 마무리하자"

트리거되지 말아야 할 케이스 (참고):
- 회사 월간 보고서 / KPI 결산 (부부 맥락 아님)
- 일반 일기 작성 (월 단위 결산이 아님)
