# Q2 — 팩트체크 AI 팀 설계

## 선택 주제
**E. 불확실한 정보 → 찾고 검증해서 최종 결론 제시**

## 워크플로우 한 줄 요약
사용자가 던진 모호한 주장/질문을 → **자료 수집 → 교차 검증 → 신뢰도 등급 포함 결론**으로 정리.

## 팀 구성 (3명)

### 1) fact-finder — 자료 수집 담당
- **역할:** 검증 대상 주장에 대해 1차 자료를 수집한다.
- **전달받는 것:** 검증할 질문 또는 주장 한 문장
- **출력하는 것:**
  - 출처 3개 이상 (URL + 발행일 + 발행 주체)
  - 각 출처에서 핵심 인용문 1~2개
  - 추측·의견 제외, 사실만 추출
- **사용 도구:** WebSearch, WebFetch

### 2) cross-verifier — 교차 검증 담당
- **역할:** fact-finder가 모은 자료를 교차 비교하고 신뢰도를 평가한다.
- **전달받는 것:** fact-finder의 출처 모음
- **출력하는 것:**
  - 출처 간 **일치/모순/불충분** 분류
  - 각 출처의 신뢰도 평가 (공식/학술/뉴스/블로그 등급화)
  - 추가 검증이 필요하면 보조 검색 1~2회 수행
- **사용 도구:** WebSearch, WebFetch

### 3) conclusion-writer — 최종 결론 정리
- **역할:** 검증 결과를 사람이 읽기 좋은 결론으로 정리한다.
- **전달받는 것:** fact-finder + cross-verifier 결과
- **출력하는 것:**
  - **판정:** `✅ 사실 / ⚠️ 부분 사실 / ❌ 거짓 / ❓ 판단 보류` 중 하나
  - 신뢰도 점수 (1~5)
  - 근거 요약 3줄 이내
  - 인용 출처 리스트
- **사용 도구:** Write (필요시 파일 저장)

## 핸드오프 흐름

```
사용자 질문
   │
   ▼
[fact-finder]  → 출처 + 인용 묶음
   │
   ▼
[cross-verifier] → 일치/모순 + 신뢰도 평가
   │
   ▼
[conclusion-writer] → 최종 판정 + 근거 요약
   │
   ▼
사용자에게 보고
```

## 설계 원칙

- **단방향 파이프라인.** 각 에이전트는 앞 단계 결과만 받아 자기 일만 한다. 역방향 의존 없음 → 메인 컨텍스트 절약.
- **추측 금지.** fact-finder와 cross-verifier는 출처 없는 진술 금지. conclusion-writer만 판정을 내림.
- **불확실하면 보류.** 모순이 크거나 출처가 부족하면 `❓ 판단 보류`로 명시. 억지로 결론 내리지 않음.
